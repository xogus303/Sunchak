# 0010. 개발 DB 호스팅: Neon(클라우드 Postgres) 공유 + Redis 로컬

- 상태: Accepted
- 날짜: 2026-07-10
- 관련: 0003(PostgreSQL), 0004(Redis)

## 맥락 (Context)
작업은 두 기기(회사/집 노트북)에서 동일하게 재현·공유돼야 한다. **코드와 DB 구조**는 git으로 이미 공유된다(집에서 clone + `prisma migrate`로 동일 재현). 그러나 **DB 데이터(실제 행)** 는 DB가 놓인 위치에 종속된다 — 로컬 Docker Postgres는 한 기기에만 존재해 데이터가 공유되지 않는다. 사용자는 "데이터까지 두 기기에서 공유"를 요구했다.

## 결정 (Decision)
- 개발 DB는 **Neon(서버리스 클라우드 Postgres)** 무료 티어에 두고, 두 기기가 **같은 `DATABASE_URL`** 로 접속해 데이터를 공유한다.
- **Redis는 각 기기 로컬**(docker-compose)로 둔다.
- 연결 문자열 등 **비밀값은 git에 올리지 않고**(.env는 gitignore) 기기별로 주입한다.
- 초기/샘플 데이터는 **시드 스크립트**(`prisma/seed`)로 재현 가능하게 한다.

## 고려한 대안 (Alternatives)
| 대안 | 장점 | 단점 / 채택하지 않은 이유 |
|---|---|---|
| **로컬 Docker Postgres만** | 무료·오프라인, 부하테스트 자유 | 데이터가 기기에 갇혀 **자동 공유 불가** — 이번 요구 불충족. |
| **Supabase** | Postgres + 인증/스토리지 부가기능 | 무료 플랜은 7일 미사용 시 일시정지. 우린 순수 Postgres면 충분. |
| **Railway** | 셋업 간편 | 상시 무료가 아님($5 체험 크레딧). |
| **Neon (채택)** | 무료 영구 티어, 유휴 시 scale-to-zero(비용 0), 순수 Postgres, Vercel 연동. 두 기기 데이터 공유. | 무료 컴퓨트·연결 한도 → **W2 대량 부하테스트엔 부적합**. |

## 근거 (Rationale)
요구가 "두 기기에서 데이터까지 공유"이므로 클라우드 공유 DB가 정답이다. Neon은 순수 Postgres에 무료·유휴 시 0원이라 학습용 공유 DB로 최적이고, 나중에 Vercel(프론트)·배포와도 자연스럽게 이어진다. Redis에 담기는 것은 캐시·대기열·원자 카운터처럼 **재생성 가능한 휘발성 상태**라 기기 간 공유가 불필요하므로 로컬로 두어 단순화한다. 비밀값을 기기별로 주입하는 방식은 **시크릿 관리**를 배우는 계기도 된다.

## 결과 (Consequences)
- 두 기기에서 같은 데이터로 작업 가능. 코드는 git, 데이터는 Neon, 비밀값은 기기별 `.env`.
- **W2 부하테스트는 로컬 Postgres로 분리**한다(무료 한도 회피, 락 동작을 왜곡 없이 관측). 같은 마이그레이션에 `DATABASE_URL`만 교체하면 되며, 세부는 W2에서 별도 정리한다.
- `.env.example`에 Neon(dev)·로컬(W2) 두 형식을 주석으로 안내한다.
